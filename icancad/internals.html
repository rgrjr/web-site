Internal documentation

Home : ICanCAD : Internals

These are my design notes, which I keep online for quick reference. Unfortunately, it's very scattershot; I tend to use internal documentation as a form of "thinking aloud," so the parts of the system with the best coverage are those that I've happened to think about more than the others. And, even for the well-documented bits, once the initial implementation is done, I tend to forget about updating the design documentation when I need to change the implementation later, as I invariably must. As a result, the code may be better at describing itself than these pages; it is safest not to trust the design documentation without also looking at the code.
