So, what is an "advanced" control structure anyway?

Home : Perl/Parrot : Advanced control structures : Introduction

Here's what I will be discussing:

  • Basic Parrot assembly code, and the runtime data structures that support bytecode execution.
  • Function calling as a control structure (e.g. map in Perl 5).
  • How to implement various sorts of nonlocal control transfers using Parrot continuations.
  • How error handling is implemented in terms of continuations.
  • How coroutines provide a thread-like interface to continuation calling.

Here's what I will not be talking about:

  • CP-transformation (another view of continuations).
  • Backtracking (see the 28th test case in t/op/lexicals.t for an example).
