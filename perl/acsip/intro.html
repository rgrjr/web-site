So, what is an "advanced" control structure anyway?

Home : Perl/Parrot : Advanced control structures : Introduction

We all know that a control structure is a way of changing the sequence of program actions. Simple control structures are implemented using goto, which jumps somewhere else in the same sub invocation. Advanced control structures involve jumps between subs. This includes such things as language constructs for exception handling. Dynamic languages support at least some of these features, which are also known collectively as nonlocal control transfers.

Here's what I will be covering:

  • Basic Parrot assembly code, and the runtime data structures that support bytecode execution.
  • Function calling as a control structure (e.g. map in Perl 5).
  • How to implement various sorts of nonlocal control transfers using Parrot continuations, with working sample code.
  • How error handling is implemented in terms of continuations.
  • How coroutines provide a thread-like interface to continuation calling.

You might be interested in this because:

  • You are writing a compiler for a language that needs these features; or
  • You write in a dynamic language, and want to know how this stuff works under the hood.

Here's what I will not be talking about:

  • Continuation-passing style and CP-transformation (another view of continuations).
  • Backtracking (see the 28th test case in t/op/lexicals.t for an example).
