The vm+qmail package

Home : Emacs tips : vm+qmail

This page describes vm+qmail, for people running vm with qmail (on a Unix or Linux system).

What are they?

qmail is a mail transfer agent (MTA) for Unix.
If your mail is delivered via qmail, then there are many neat things you can do. Here are two of them:
  1. You can subscribe to mailing lists using distinct return addresses so that mail from each list is delivered to a separate inbox.
  2. You can use Maildir format for your inboxes, instead of the traditional Unix mbox format that suffers from the standard concurrency bugs.
Not all mail readers handle these features well, though.

The vm package allows you to read mail within emacs.
vm is a modern MIME-aware mail-reading program (also called a mail user agent or MUA) with many powerful features, in addition to the convenience of being integrated into the emacs editor.
  1. vm allows you to assign a distinct inbox to each of several folders, but then it's harder to know whether you have new mail waiting.
  2. vm version 6.75 doesn't handle Maildir format natively.

vm+qmail improves integration between the two.

Commands

  • M-x v+q-mbox-status reports on any incoming messages currently sitting in mailbox files or Maildir directories (so the command name is really a bit of a misnomer). The set of files/directories examined is controllde by the v+q-mbox-directory and v+q-mbox-regexp variables, which default to "~/" and "^Mailbox\\|^Maildir", respectively. If a directory is matched, it is assumed to be a Maildir; otherwise, it is assumed to be an mbox-format file. In either case the total number of messages are counted, and summarized in a line that looks like: 
      1 message in ~/Mailbox, 13 in ~/Mailbox.bugtraq, 8 in ~/Maildir.netatalk
    Messages are reported regardless of whether the box or directory appears as an inbox in the vm-spool-files list. Doesn't handle POP or IMAP drops.

  • M-x v+q-generate-password generates a password using alphanumerics plus unshifted special characters. It will be echoed in the message area; see the *Messages* buffer if you want to cut-and-paste it somewhere. The default length is 8 characters long, unless you give the command a numeric argument. If the argument is negative, it will produce something that doesn't use shift keys at all (i.e. no uppercase letters). [This is part of vm+qmail for no particular reason. -- rgr, 9-Jul-00.]
