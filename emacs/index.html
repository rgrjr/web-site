Notation conventions

I use an HTML-ized version of the standard emacs notation:
  • find-file is the name of an emacs lisp function (the same style is used for variables);
  • C-x C-f denotes the standard key sequence that invokes find-file, but
  • M-x find-file also works to invoke this command.
  • "C-x C-f foo TAB bar TAB RET" denotes a specific sequence of characters that invokes one or more commands, possibly with arguments. The spaces in this string are just for readability and are not to be typed; if a space character is required, it will be written SPC. This example uses 11 characters (C-x, C-f, f, o, o, TAB, b, a, r, TAB, and RET); the enclosing quotes don't count.
  • In a defining list of commands,
    • C-x C-f (find-file)
    shows both the standard key sequence, and the command it invokes in parentheses, followed by a description of what it does. Since different editing modes can use the same key sequence for different commands, it is unambiguous to include both. (But command names are usually omitted for keys that are rarely rebound.)

Learning emacs

The ideal way to learn emacs, really the only way, is to sit down in front of it and bang away. If you have access to emacs, then you have several options:
  • Use the built-in emacs tutorial. Simply start emacs and type C-h t; follow instructions from there.
  • Look through the "Other resources" pages below, bring your favorite up in a browser next to emacs, and experiment.
  • Or, get one of the O'Reilly emacs offerings mentioned below, and start playing.
In your experimentation, be sure to edit source code in Tcl, or a language you know.

If you don't have access to emacs, then your options are much more limited. I emphatically don't recommend trying to learn emacs solely from a book; you'll only put yourself to sleep. If you can't download and install it yourself on your machine, then you're best off glancing at, say, the emacsis.great tutorial page, and otherwise waiting until you can get your hands on a keyboard.

Other resources for learning emacs

Cheat sheets and other short summaries:

  • Dave Cohen's emacs cheat sheet, written by an alumnus of the August 1999 bootcamp.
  • Also, Bob Rogers' extended version of Dave Cohen's emacs cheat sheet.
  • Tutorial outline for the tutorial presented at bootcamp on 4-Oct-99.
  • www.refcards.org has an information-rich emacs reference card (which doesn't seem to want to print on printers of vintage 1995 or earlier).

Tutorials and other longer articles:

  • The Linux Emacs Beginner's HOWTO by Jeremy D. Zawodny gives a good overview of emacs that is not specific to Linux. It is available from http://www.wcnet.org/jzawodn/emacs/ as well as from http://www.linuxdoc.org/HOWTO/Emacs-Beginner-HOWTO.html.
  • emacsis.great tutorial page (not hypertext), with a topical command summary at the bottom. This is very dense, and covers a lot of ground.
  • GNU emacs FAQ. This is also available in emacs itself by typing "C-h F" (note that the "F" must be capitalized). The Web version is hypertext, though it isn't very easy to navigate, and it does have a reference page that points to all kinds of good stuff. The version within emacs is not hypertext, but it's all on one page, and you can use C-s (incremental search) to find things of interest.
