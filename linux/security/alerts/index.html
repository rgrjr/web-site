Security alerts

Home : Linux : Security : Alerts

Every now and then, when I learn of some severe security problem that impacts a large number of computer users, I decide to send email to alert my friends and acquaintances. On a somewhat more frequent basis, I send alerts concerning server security problems to a group of fellow Linux users. I have started collecting these online, for the record, and for easy reference.

None of these are original discoveries, but are summaries of security advisories posted elsewhere, usually on bugtraq. The alerts in the first batch are of general interest, and are aimed at the non-expert. Those in the second batch are focused toward Linux users who run servers, but are usually also of interest to Unix sysadmins generally.

Alerts of interest to people running clients

This means you!

Note that most of these also affect Windows users; they are filed under "/linux/security/" for historical reasons.

"Some excellent reasons to ditch Internet Explorer, Flash" (14 Aug 2002)
Omnibus alert that discusses (1) a severe SSL cert-checking vulnerability in MSIE and Konqueror (sometime referred to as "certificate chain spoofing"), (2) the random unpatched IE vulnerabilities listed at http://www.pivx.com/larholm/unpatched/, (3) an exploitable Shockwave Flash buffer overrun, and (4) the new licensing terms for Windows Update and Automatic Update that one must accept to get Windows 2000 SP3. On 2 September, it was revealed that Outlook is also affected.
"MSIE download dialog may lie about file type (26 Nov 2001)
"Followup to: MSIE download dialog may lie about file type (16 Dec 2001)
Excerpts two advisories about MSIE problems that allow Web sites to run arbitrary programs on a user's computer with inadequate or no warning. Microsoft has said that they do not consider this a problem.

Alerts of interest to people running servers

"Trojaned OpenSSH on OpenBSD server" (1 Aug 2002)
This is no longer relevant; somebody managed to slip a trojan into the OpenSSH tarfile, but it was detected quickly. I posted this in case somebody downloaded a fresh copy to rebuild after updating OpenSSL.
"Multiple OpenSSL problems" (30 Jul 2002)
Multiple remotely exploitable vulnerabilities in OpenSSL pre-0.9.6d; used by OpenSSH, among others.
"[Theo de Raadt: Upcoming OpenSSH vulnerability]" (26 Jun 2002)
"[Theo de Raadt: Upcoming OpenSSH vulnerability]" (followup; 26 Jun 2002)
Remotely exploitable vulnerabilities in OpenSSH pre-3.3.1.
"'Code Red' worm" (20 Jul 2001)
Informational message about the original "Code Red" worm, which affects Microsoft IIS Web servers. Note that "Code Red II," which came to light a few months later, is much more severe. A year after CRII, I still see hits from these worms in my Web logs, indicating unpatched servers.
"Serious Linux kernel flaw; upgrade now!" (5 Jul 2000)
Conceptual bug in Linux "capabilities" that makes setuid programs readily exploitable. (Affects all Linux users, but is a local exploit, so requires shell access -- or an exploitable server bug.)
