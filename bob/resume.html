Employment history

ESS Technology, Kelowna BC Design Centre (satellite office), May-01 to Mar-02.

  • Continued development of the ICanCAD schematic editor, bringing it to full schematic editing functionality. ICanCAD is focused on mixed-signal IC circuit design (i.e. both analog and digital signals are "mixed" on the same integrated circuit). ICanCAD 0.2.0 boasts the following features:
    • Rich visual feedback during schematic entry.
    • An "Undo/Redo" stack of arbitrary depth.
    • Support for parameter passing from hierarchical parent to child. A single schematic can thus be used to represent a whole family of layouts that share a common circuit topology.
    • A special "speed wiring hack" for wiring many series connections with a single click.
    • Plus a number of other special features to speed design entry.
    Release 0.2.0 consists of nearly 27K lines of Common Lisp source, plus 10K lines of HTML documentation.
  • Extended the Discus package (see below) to support secure symmetric directory mirroring via rsync or scp.
  • Built my own office network (wiring, hardware, and software) to support a dual boot Linux/Windows PC and an AlphaStation DS10 running Tru64 Unix, with the virtual Open Genera 2.0 system on top of Tru64. The network has four computers running a total of five operating systems (six, if you count Red Hat Linux as distinct from SuSE Linux). The main machine runs number of servers to support the various satellite machines (details below).
  • Studied Virtual Private Network (VPN) technologies, especially FreeS/WAN.
  • Provided Linux, X11, and OpenSSH support to Kelowna and the Fremont CA headquarters.

BioMolecular Engineering Research Center (BMERC), Boston University, Feb-94 to May-01.

  • Protein Structure Analysis (PSA) server for protein structure prediction using designed Hidden Markov Models (HMMs); nearly 40 000 protein structure predictions have been made since Fall 1994. Designed and wrote the mail server in emacs lisp as a front end to a structure analysis tool written by J. V. White and C. M. Stultz in the Matlab language. Took over maintenance of the Web interface in early 1998. Assumed front-line responsibility for maintaining the Matlab code in late 1999 (and wrote a dot-m-file parser in Common Lisp in order to analyze the code). Consists of 6.3K lines of emacs lisp (the server proper), 1.7K lines of perl (server support and CGI interface), 4.6K lines of Matlab .m files, and a few hundred lines of miscellaneous shell scripts.
  • Codeveloper of the needle protein structure prediction toolkit. 42K lines of Common Lisp (the majority of which was originally designed and written by Richard H. Lathrop), plus 2K lines of C.
  • Wrote the needle tools software suite for manipulation of and extraction of statistics from entries in the Protein Data Bank, a flat-file protein structure database; used as support software for needle. 7.7K lines of perl, 9K lines of C, 600 lines of csh scripts, and 13.8K lines of HTML documentation. (Much of the C code was inherited, but has been completely rewritten in the process of extension.) The source code for all scripts (perl and csh) is browsable at http://bmerc-www.bu.edu/information/script_frame.html. See also http://bmerc-www.bu.edu/needle-doc/new/perl-sub-doc.html.
  • Wrote the Discus package for symmetric directory mirroring via FTP. Discus consists of 5K lines of emacs lisp, and has been used to support the distributed development of needle since 1994.
  • Designated site emacs wizard. Responsible for emacs upgrades on the three principal BMERC system types (SunOS 4.1.4, Solaris 7, Tru64 4.0). Wrote more than 10K lines of emacs lisp to support HTML editing and various other uses, some of which is published as rgr-hacks.
  • Assisted our sysadmin while she installed netatalk on two Solaris servers. Designer and principal implementer of our site-wide qmail upgrade strategy. Advisor on security issues; shock-trooper on security emergencies, such as the wu-ftpd "SITE EXEC" fiasco.

Silicon Analog Systems, Kelowna BC, Canada, part-time consulting during Jan-00 through Mar-01.

  • Developed a prototype analog IC CAD system in Common Lisp. After seven weeks (FTE) of design and implementation work as of Mar-01, ICanCAD 0.1.3 consists of about 9K lines of Common Lisp and 3400 lines of HTML documentation. ICanCAD 0.1.3 is capable of viewing existing schematics (produced by a legacy CAD system), and is starting to show some limited editing functionality. Work has been done on producing EDIF Version 2 0 0 Level 0 output, but a complete implementation requires full electrical connectivity semantics. ICanCAD runs on Allegro Common Lisp using CLIM graphics, and on CMU Common Lisp (without graphics).

ArsDigita Web/Database Bootcamp, Westwood MA, Oct-99.

  • Principal TA for the three-week Web/Database Developers' Bootcamp offered by ArsDigita at that time. This was a shorter version of the MIT course taught by Philip Greenspun, the ArsDigita founder. (I had just taken this bootcamp myself in Aug-99.) ArsDigita was acquired by Red Hat in February 2002.
  • I gave brief seminars on basic emacs usage and Tcl regular expressions, and helped people design and debug their Tcl, SQL, and HTML as they worked their way through the problem sets.
  • Wrote some emacs help pages, plus general pre-Bootcamp study materials.

Analog Devices, Linear Signal Processing (LSP) division, Wilmington MA, Jan-86 through May-92.

  • Janus CAD system for mixed-signal IC design and layout. Contributed extensively to the layout and simulation interface subsystems; rewrote the common UI and library management components. The system as a whole was roughly 150K lines of Zetalisp/Common Lisp using Flavors (an object-oriented language extension) on Symbolics Lisp Machines. Software was internally distributed within Analog.
  • "New Janus" CAD system (the next-generation version of Janus). Roughly 80K lines of Common Lisp using Flavors/CLOS (more OO extensions) and CLIM (Lisp GUI toolkit) developed on Symbolics and ported to Sun Sparcstations running Allegro Common Lisp. Architect, project technical leader, and principal implementor.
